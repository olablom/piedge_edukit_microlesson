PiEdge EduKit — Guided Demo (Micro-lesson)

Goal: Run a 30-min self-contained Edge-ML pipeline, generate artifacts and a progress/receipt.json proving completion.

Prereqs

  • Python 3.12 installed
  • Jupyter Lab/Notebook (optional for interactive)
  • OS: Ubuntu 22.04 / macOS / Windows 11

Environment template: .env.example (no vars needed; included for packaging spec)

Setup (.venv)

Linux/mac:

bash setup_venv.sh
source .venv/bin/activate

Windows (PowerShell/VS Code Terminal):

py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Run the lesson

Linux/mac:

bash run_lesson.sh

Windows:

run_lesson.bat

This runs the full pipeline headless and opens the HTML report.

Interactive mode

source .venv/bin/activate   # or .\.venv\Scripts\activate
python main.py --lab         # opens Jupyter Lab

Open notebooks/00_run_everything.ipynb and run all cells.

What you'll produce

  • models/model.onnx
  • reports/*.json/png
  • notebooks/run_everything_REPORT.html (auto-opened)
  • progress/receipt.json (PASS/FAIL + metrics)

Verify & receipt

Re-run verification at any time:

python verify.py \
  --lat-fp32 reports/latency_summary_fp32.json \
  --lat-int8  reports/latency_summary_int8.json \
  --acc-json  reports/accuracy_for_verify.json \
  --receipt   progress/receipt.json \
  --progress  progress/lesson_progress.json \
  --min-speedup-pct -10.0

Troubleshooting

  • If INT8 quantization fails, FP32 fallback is accepted (receipt still PASS).
  • If images don't show in the HTML, ensure notebooks/setup_diagram.png exists.
